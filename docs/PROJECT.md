# Trakmile — وثائق المشروع

## نظرة عامة

هذا الموقع هو **موقع تعريفي** لمنصة **Trakmile** (تراك مايل) — منصة إدارة لوجستية ذكية.  
المشروع بسيط حالياً ويتكون من:

- صفحة تعريفية واحدة (`index.html`)
- واجهات API لربط **تطبيق الجوال** (مجلد `mobile-api`)
- وثائق النظام ونسخة من قاعدة البيانات (مجلد `docs`)

## تطبيق الجوال (`mobile-api`)

تتواصل نقاط API مع تطبيق React Native عبر **WebView**، وترسل الردود بـ `window.ReactNativeWebView.postMessage()`.

### آلية العمل

1. التطبيق يرسل طلباً عبر GET إلى أحد ملفات PHP داخل WebView مخفية.
2. السكربت يتحقق من **النطاق الفرعي** (`Sub_Domain`) في جدول `domains`.
3. عند نجاح التحقق، يُفتح اتصال بقاعدة بيانات العميل (`DB_Name`, `DB_User`, `DB_Pass`) ويُنفَّذ الإجراء المطلوب.
4. النتيجة تُعاد كـ JSON إلى التطبيق.

### الملفات الرئيسية

| الملف | الوظيفة |
|-------|---------|
| `check_domain.php` | التحقق من صحة النطاق الفرعي وتاريخ انتهاء الاشتراك |
| `login.php` | تسجيل دخول المستخدم وربط FCM token |
| `home.php` | بيانات لوحة التحكم في التطبيق |
| `getOrders.php` / `orders.php` | جلب وعرض الطلبات |
| `confirmOrderApi.php` | تأكيد الطلب |
| `update_password.php` | تحديث كلمة مرور المستخدم |

## قاعدة البيانات

نسخة SQL موجودة في: `docs/DB/trak_db.sql`

### قاعدة البيانات الرئيسية: `trak_db`

تحتوي حالياً على **جدول واحد** لربط تطبيق الجوال:

#### جدول `domains`

يخزّن بيانات ربط كل عميل (نطاق فرعي) بقاعدة بياناته الخاصة على المنصة.

| العمود | الوصف |
|--------|-------|
| `ID` | المعرّف |
| `Domain` | النطاق الرئيسي (مثال: trakmile.com) |
| `Sub_Domain` | النطاق الفرعي للعميل |
| `DB_Name` | اسم قاعدة بيانات العميل |
| `DB_User` | مستخدم قاعدة البيانات |
| `DB_Pass` | كلمة مرور قاعدة البيانات |
| `Token` | رمز التحقق للتطبيق |
| `c_code` | رمز الشركة |
| `expire_date` | تاريخ انتهاء الاشتراك |

> **ملاحظة:** بيانات العملاء الفعلية (الطلبات، المستخدمون، إلخ) تُخزَّن في قواعد بيانات منفصلة لكل عميل، وليست في `trak_db`.
